# VoiceOrder 웹 기획 명세서

## 1. 프로젝트 개요

- 프로젝트명: VoiceOrder
- 한줄 소개: 음성 인식 기반으로 메뉴를 빠르게 주문할 수 있는 웹 키오스크 데모
- 프로젝트 목적: 터치 입력과 음성 입력을 함께 지원해 주문 속도와 접근성을 높이는 주문형 웹 서비스 구조를 검증한다.

## 2. 기획 주제

이 프로젝트의 핵심 주제는 "음성 주문이 가능한 웹 키오스크"이다.  
사용자는 일반 키오스크처럼 메뉴를 직접 선택할 수도 있고, 마이크를 통해 메뉴명과 수량을 말해 장바구니를 구성할 수도 있다.

특히 다음과 같은 상황을 고려한 서비스 방향을 가진다.

- 매장 키오스크에서 빠른 주문이 필요한 경우
- 고령층 또는 터치 입력이 불편한 사용자를 보조하는 경우
- 향후 AI 음성 주문 인터페이스로 확장 가능한 구조를 시험하는 경우

## 3. 주요 대상 사용자

- 카페, 패스트푸드 매장 방문 고객
- 키오스크 사용이 익숙하지 않은 사용자
- 음성 UI 기반 주문 경험을 테스트하려는 기획자 및 개발자

## 4. 핵심 기능

### 4-1. 메뉴 조회

- 서버에서 메뉴 목록을 불러와 카드 UI로 출력
- 메뉴명, 가격, 이미지 정보를 함께 제공
- 사용자는 클릭만으로 장바구니에 메뉴를 추가 가능

### 4-2. 음성 주문

- 브라우저의 Web Speech API를 통해 음성을 텍스트로 변환
- 예: "아이스 아메리카노 두 잔", "케이크 하나 주세요"
- 인식된 텍스트를 서버로 전송해 주문 후보를 분석

### 4-3. 주문 문장 해석

- 서버에서 문자열 기반 메뉴명 매칭 수행
- 메뉴 별칭 처리 지원
  - 예: 아메리카노, 아아, 라떼, 케이크 등
- 한국어 수량 표현 일부 지원
  - 예: 한, 두, 세, 열, 2잔, 3개 등

### 4-4. 장바구니 관리

- 수동 추가와 음성 주문 결과를 하나의 장바구니에 통합
- 동일 메뉴는 수량 누적
- 수량 증가/감소 및 장바구니 비우기 가능

### 4-5. 주문 저장

- 결제하기 버튼 클릭 시 주문 데이터를 DB에 저장
- 주문 번호, 총 결제 금액, 주문 시간을 반환

## 5. 사용 기술 스택

### 5-1. 언어

- Python
- HTML
- JavaScript
- SQL

### 5-2. 프레임워크 및 라이브러리

- Flask

- SQLite
  - 로컬 개발용 데이터베이스

## 6. 기능별 사용 툴 및 기술 정리

| 구분 | 구현 내용 | 사용 기술/툴 |
|---|---|---|
| 웹 서버 | 페이지 렌더링, API 제공 | Flask |
| 데이터 저장 | 메뉴/주문/주문 상세 저장 | SQLite |
| DB 초기화 | 테이블 생성 및 샘플 메뉴 입력 | Python, sqlite3 |
| 프론트 UI | 키오스크 메인 화면, 장바구니, 음성 주문 패널 | HTML, Tailwind CSS, JavaScript |
| 음성 인식 | 마이크 입력을 텍스트로 변환 | Web Speech API |
| 주문 분석 | 메뉴명/수량 문자열 매칭 | Python 정규식, 사전 매핑 |
| 비동기 통신 | 메뉴 조회, 음성 주문 전송, 주문 저장 | Fetch API |

## 7. 화면 구성 요약

### 메인 화면

- 상단: 서비스 소개, 음성 상태, 인식 텍스트, 총 금액
- 좌측: 메뉴 카드 목록
- 우측 상단: 음성 주문 영역
- 우측 하단: 장바구니 및 결제 영역

### 사용자 흐름

1. 사용자가 메뉴를 직접 누르거나 음성 주문 버튼을 누른다.
2. 음성 주문 시 브라우저가 마이크 권한을 확인한다.
3. 인식된 텍스트가 서버로 전송된다.
4. 서버가 메뉴와 수량을 해석해 장바구니 후보를 반환한다.
5. 사용자는 장바구니를 확인 후 결제 버튼을 누른다.
6. 주문 정보가 DB에 저장된다.

## 8. 현재 구현 기준 장점

- 구조가 단순해 데모, 발표, 포트폴리오 용도로 적합함
- 프론트와 백엔드가 분리되어 있어 기능 확장이 쉬움
- 음성 주문이라는 차별 요소가 명확함
- SQLite 기반이라 별도 서버 구성 없이 빠르게 실행 가능함

## 9. 보완이 필요한 부분

### 9-1. 보안

- 현재는 로그인/권한 관리가 없음
- 주문 API에 대한 인증 절차가 없음
- CSRF, 요청 위조 방어 구조가 없음
- 입력 검증은 기본 수준이라 악의적 요청에 대한 방어 강화 필요
- `debug=True` 실행은 운영 환경에서 비활성화 필요
- SQLite 파일이 로컬에 직접 저장되므로 운영 환경에서는 분리된 DB 서버 사용 권장

### 9-2. 기능 정확도

- 음성 주문 해석이 단순 문자열 매칭이라 복잡한 문장 처리에 약함
- 동의어 사전이 제한적이라 실제 사용자 발화 다양성을 충분히 반영하지 못함
- 숫자 표현 해석 범위가 제한적임
- 여러 메뉴가 한 문장에 섞인 경우 정교한 파싱 개선 필요

### 9-3. 운영 관점

- 관리자 페이지가 없어 메뉴 관리와 주문 조회가 불편함
- 결제 시스템 연동이 없어 실제 결제 서비스로는 미완성 상태
- 주문 취소, 영수증, 재주문, 매출 통계 기능이 없음

### 10-2. 서비스 기능 확장

- 포인트 적립
- 매장 식사 / 포장 선택
- 옵션 선택
  - 예: 샷 추가, 얼음 양, 사이즈 변경
- 주문 내역 조회 및 재주문
- 관리자용 메뉴 등록/수정/삭제 기능

### 10-3. 기술 확장

- REST API 고도화

- 클라우드 환경 배포
  - 예: Vercel 프론트, Render 또는 Railway 백엔드, Supabase DB

## 13. 한줄 요약

VoiceOrder는 Flask와 SQLite를 기반으로 구현한 음성 주문 웹 키오스크 데모이며, 현재는 프로토타입 단계지만 주문 UX, 접근성, 음성 인터페이스, AI 확장성 측면에서 발전 가능성이 높은 구조를 갖고 있다.
